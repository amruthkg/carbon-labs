## Overview

**Time Display** provides an explicit, accessible, Carbon-aligned way to show
elapsed time, remaining time, or a fixed duration using configurable units,
structured display formats, and subtle productive-motion digit transitions.

It is intended for **duration-based time** — not wall-clock date/time display.

## Getting started

```bash
yarn add @carbon/react
yarn add @carbon-labs/react-time-display
```

### JS (via import)

```jsx
import { TimeDisplay } from '@carbon-labs/react-time-display/es/index';

function App() {
  const jobStart = new Date(Date.now() - 3665000);
  return (
    <TimeDisplay>
      <TimeDisplay.Label>Job elapsed time</TimeDisplay.Label>
      <TimeDisplay.Value mode="count-up" startTime={jobStart} format="stacked" />
      <TimeDisplay.HelperText>Job is still running</TimeDisplay.HelperText>
    </TimeDisplay>
  );
}
```

## Composable API

`TimeDisplay` uses a composable sub-component pattern. The root `<TimeDisplay>`
provides the shared context; child components control layout, content, and order.

```jsx
<TimeDisplay animated>
  {/* Label — plain string preferred for automatic SR text assembly */}
  <TimeDisplay.Label>Elapsed time</TimeDisplay.Label>

  {/* Value — owns the timer; mode is required */}
  <TimeDisplay.Value
    mode="count-up"
    startTime={start}
    format="stacked"
    units={['hours', 'minutes', 'seconds']}
  />

  {/* Optional helper text */}
  <TimeDisplay.HelperText>Job is still running</TimeDisplay.HelperText>

  {/* Optional completion message — consumer mounts conditionally */}
  {isComplete && (
    <TimeDisplay.CompleteMessage>Job complete</TimeDisplay.CompleteMessage>
  )}
</TimeDisplay>
```

Because sub-components are plain children, order and custom content are fully
under consumer control:

```jsx
<TimeDisplay>
  <TimeDisplay.Value mode="count-down" endTime={deadline} format="flat" />
  <TimeDisplay.Label>Time remaining</TimeDisplay.Label>   {/* label below value */}
  <div className="my-progress-bar" />                     {/* custom content */}
  <TimeDisplay.HelperText>Save your work</TimeDisplay.HelperText>
</TimeDisplay>
```

## Formats

Four display formats are available:

<Canvas of={TimeDisplayStories.AllFormats} />

| Format | Description |
|---|---|
| `stacked` | Large numeric values with unit labels below. Default. |
| `boxed` | Each unit in a flat Carbon-style box with a label below. |
| `flat` | Compact separator-delimited string, e.g. `01:23:45`. |
| `inline` | Dense inline text, e.g. `1 hr 23 min 45 sec`. |

## Component API

`TimeDisplay` is composed of five sub-components. Use the **tabs** in the
Component API panel below to browse each sub-component's full prop table.

| Sub-component | Purpose |
|---|---|
| `TimeDisplay` | Root — context provider, layout, `animated` modifier |
| `TimeDisplay.Label` | Visible label; registers plain-string text for SR assembly |
| `TimeDisplay.Value` | Timer owner — all four formats, validation, SR write-back |
| `TimeDisplay.HelperText` | Supplementary text below the value |
| `TimeDisplay.CompleteMessage` | Completion message — consumer mounts conditionally |
